## **1. 引言**
### 1.1 背景与目标
在高等教育数字化转型中，标准化教学难以适配学生个体差异，学习资源繁杂且缺乏个性化引导。本系统基于大模型与多智能体技术，以某一具体专业课程（如"大语言模型原理与实践"）为切入点，构建一个可自动生成个性化学习资源、规划自适应学习路径的智能学习环境。其核心目标是实现"因材施教"的数字化落地，让学生以类似探索"藏宝图"的方式，在动态调整的关卡化学习中获得适配自身水平的教学资源。

### 1.2 产品定位
一款**桌面式多窗口个性化学习工作站**。它摒弃传统填鸭式课程表，将AI生成的"学习藏宝图"作为导航中枢，通过多个可自由拖拽的功能窗口，为学生提供讲义阅读、习题演练、AI对话辅导、代码实验等沉浸式学习体验。系统以多智能体协同驱动，学习资源即时生成、路径动态演化，完整覆盖学习闭环。

### 1.3 典型用户故事
> 小明是一名计算机专业大三学生，目标是掌握大语言模型技术。他打开系统，与AI进行了5分钟的对话，系统构建出他的初始画像——数学基础较好，但注意力易分散，偏好图形化学习。AI随即生成了一张"大语言模型学习藏宝图"，从"词嵌入"到"Transformer"再到"RLHF"，层层递进且存在可选分支。小明在左侧藏宝图中右键单击"自注意力机制"关卡节点，系统自动为他打开了讲义窗口、思维导图窗口和习题窗口——窗口已排好布局，无需任何配置。讲义图文并茂，导图可交互展开，习题分三个难度。他发现一个好奇的方向，于是将自己的论文笔记PDF上传至"我的资源库"，拖拽到当前关卡节点完成绑定，AI随即在该关卡的拓展阅读中优先引用了他的资料。学习完成后，画像中的"知识基础"维度上升了，藏宝图为他解锁了一个"视觉Transformer兴趣分支"。小明决定探索这个新分支，整个学习过程如同探险。
>
> 小丽正在复习大语言模型知识，她切换到"探索模式"，系统不再提升关卡难度。她右键一个关卡节点，系统识别为理论型关卡，自动打开讲义、导图和习题三个窗口。她觉得导图窗口占了太多空间，直接点击"×"关闭，系统记住了这个偏好。练完这套题，她点击顶部"✏️ 刷题模式"图标，所有窗口瞬间切换到刷题布局——习题居中放大，AI对话移到右侧——内容进度完全保留。

## **2. 系统功能需求**
### 2.1 对话式学习画像自主构建
**功能描述**
通过自然语言对话采集学生信息，自动抽取特征构建动态画像，并在学习过程中持续更新。

**核心流程**

1. **对话引导**：由**画像 Agent** 主动发起对话，以轻松问答形式了解学生的专业、学习目标、历史学习经历、自我感知的薄弱点、偏好的学习方式等。
2. **特征抽取**：从对话文本中提取至少**6个画像维度**：
    - 知识基础（已具备的先修知识）
    - 学习风格（视觉/文字/实操偏好）
    - 认知风格（分析型/直觉型）
    - 目标导向（考研/就业/兴趣拓展）
    - 易错点偏好（易在哪些类型题目犯错）
    - 专注耐性（长时间/碎片化学习）
3. **画像展示**：在界面中常驻一个**画像卡片**窗口，以雷达图、维度条等形式直观展示当前画像。
4. **随学随新**：学生每完成一个学习关卡或进行一段辅导后，系统根据其答题表现、提问类型、资源浏览时长等数据，自动更新画像维度数值，卡片产生可见的数值变化动画——对应维度闪光并伴随数值增长动画（0.5秒内完成），学生可直观感知自身进步。
5. **画像编辑**：画像卡片上提供"编辑"按钮，学生可手动调整某个维度的标签或数值（如将"注意力分散"改为"注意力集中"），系统同步更新画像。

**画像数据结构示例**

```json
{
  "student_id": "s1001",
  "dimensions": {
    "knowledge_base": 0.65,
    "visual_preference": 0.8,
    "analytic_style": 0.5,
    "goal_employment": 0.9,
    "error_prone_type": "conceptual",
    "focus_duration": 25
  },
  "tags": ["Python熟练", "数学较好", "注意力分散"],
  "last_updated": "2026-06-07T10:00:00"
}
```

### 2.2 多分支"学习藏宝图"与动态路径规划
**功能描述**
依据学生画像，由规划 Agent 生成一张**非线性、可演化的知识关卡网络**（藏宝图），为学生提供全局学习路径视图，并根据学习反馈实时调整。

**核心功能要求**

1. **初始生成**
    - 学生设定学习科目与目标后，**规划 Agent** 结合画像分析，生成初始藏宝图。
    - 藏宝图以**知识图谱形式**组织，每个节点代表一个学习关卡（聚焦一个知识点），节点间的有向边表示前置依赖或推荐顺序。
    - 必须包含**分支结构**：例如学习完"RNN"后，可分支到"LSTM"或"GRU"，亦可有"传统机器学习基础巩固"的可选分支。
2. **关卡设计**
    - 每个关卡明确标注：知识点名称、难度预估、建议学习时长、依赖关卡。
    - 关卡分为**核心路径**（必须完成）和**兴趣分支**（可选探索），视觉上以不同颜色区分。
3. **自适应演化机制**
    - **难度调整**：依据当前学习模式（见下文"模式切换"）决定难度策略：
      - **精通模式（🎯）**：若学生快速通过某关卡，后续同类型关卡的难度自动提升；若频繁出错，则自动插入基础强化节点或降低预期难度。
      - **探索模式（🔭）**：系统绝不提升难度，所有关卡按标准难度生成，适合快速了解、复习或初步学习。
    - **模式切换**：桌面顶部状态栏提供全局学习模式切换开关，默认首次为探索模式，用户可随时手动切换。切换模式只影响未来新解锁的关卡，已生成的关卡内容及难度不变。突击模式（见下文）强制采用探索模式的难度策略。
    - **动态插入**：当系统检测到学生对某一知识点关联领域有持续提问（如注意力机制在图像中的应用），可自动生成一个"兴趣分支"关卡链，加入藏宝图。
    - **突击模式**：学生可主动输入目标（如"三天攻克softmax推导"），规划 Agent 会打破原有顺序，从依赖图中抽取最精简的关键路径，生成临时强化序列。突击模式强制采用探索模式的难度策略，避免突击学习时难度突增。
4. **被放弃分支的处理（分支休眠）**
    - 当用户长期未访问某兴趣分支，或主动选择"放弃此分支"时，该分支的连线和节点变为浅灰色虚线，节点半透明并显示小的"休眠"图标（💤）。
    - 鼠标悬停时提示："此分支已休眠，可右键点击重新激活"。
    - 右键单击休眠节点，弹出菜单"重新激活分支"，系统将其恢复为正常兴趣分支（银灰色实线 / 空心节点）。
    - 分支被放弃后不会删除，仍保留在地图上，但默认不干扰主线视野。
5. **节点全局移动**
    - 用户可长按鼠标左键（超过0.5秒）并拖动任意节点，改变其在画布上的视觉位置。
    - 此操作仅改变节点的布局坐标，不改变知识图谱中的依赖关系（连线始终保持连接）。
    - 支持配合 `Ctrl` 键多选节点后批量拖拽。
    - 目的：允许用户自定义藏宝图的视觉布局，方便聚焦重点或整理个人学习路径。
6. **可视化与交互**
    - 藏宝图在独立窗口中渲染，支持缩放、拖拽、节点点击查看详情。
    - 已完成节点打勾，当前解锁节点高亮，未解锁节点置灰但可见全局。
    - 鼠标悬停节点显示简介，点击进入该关卡学习界面。

### 2.3 多智能体协同的关卡内即时资源生成
**功能描述**
当学生进入某一关卡时，由多个内容生成 Agent 即时协作生成该关卡的专属学习材料，类型不少于5种，保证资源的新鲜度与针对性。

**协同流程**

```plain
规划Agent 指定关卡知识点
       ↓
关卡生成总监Agent 制定该关卡的资源列表
       ↓
┌───────┼───────┬───────┬───────┐
文档Agent  导图Agent  习题Agent  代码Agent  (视频Agent 加分项)
       ↓
评审Agent 交叉校验内容质量与学术准确性
       ↓
资源打包呈现给前端界面
```

所有生成过程通过消息队列传递，前端展示**流式生成进度条**，避免长时间白屏等待。

**必选资源类型**

1. **核心讲义文档**：图文并茂的知识点讲解，支持Markdown渲染，包含公式、图表、关键概念高亮。
2. **知识结构导图**：以 `.md` / `.mm` 格式生成思维导图，可交互展开/折叠，展示概念的层级关系。
3. **分级练习题组**：包含基础巩固、进阶应用、挑战思考三个难度等级，支持自动批改与解析生成。
4. **拓展阅读材料**：关联经典论文摘要、技术博客导读、视频教程链接等，附AI生成的推荐理由。若该关卡已绑定用户上传的资源，拓展阅读中将优先引用用户资料并生成个性化的讲义或习题。
5. **实操案例/代码实验**：针对编程相关知识点，生成Jupyter Notebook或完整代码片段，可在内置代码运行环境中直接执行。

**加分项（选做）**
6. **多模态教学视频/动画**：对抽象概念（如反向传播、注意力机制）生成1-2分钟的动画或虚拟讲师讲解视频，支持播放与暂停。

**质量保障：评审Agent（防幻觉）**

+ **职责**：实时审查生成内容的事实一致性、学术准确性、敏感信息安全。
+ **机制**：采用"检索增强生成（RAG）+ 交叉验证"策略。评审Agent访问内置的权威知识库（教材、论文、百科），对生成内容逐条标注引用，删除未证实或可能产生误导的陈述。
+ **违规处理**：若检测到潜在错误或敏感词，自动重新生成或回退至安全版本，并记录日志。

### 2.4 多窗口沉浸式学习与智能辅导
**功能描述**
提供类似桌面操作系统的多窗口学习环境，学生可自由组织学习界面，并在任何情境下唤起AI辅导。

#### 2.4.1 核心应用窗口清单
系统提供以下9个核心应用窗口，覆盖完整学习闭环：

| # | 应用窗口 | 说明 |
| --- | --- | --- |
| 1 | 藏宝图 | 知识关卡导航，支持缩放、拖拽、节点操作 |
| 2 | AI对话悬浮窗 | 可浮动或固定到某一侧，上下文感知辅导 |
| 3 | 讲义阅读器 | Markdown渲染的关卡内容阅读 |
| 4 | 习题练习面板 | 分级练习、自动批改、错题本 |
| 5 | 代码编辑器 + 运行终端 | 代码实验与运行结果查看 |
| 6 | 笔记本/批注工具 | 个人笔记与内容标注 |
| 7 | 思维导图查看器 | 知识结构导图展开/折叠 |
| 8 | 我的资源库 | 用户上传资料的管理与绑定 |
| 9 | 学习仪表盘 | 学习统计与画像演变可视化 |

#### 2.4.2 桌面式工作区

+ 系统主界面为一个"桌面"，可同时打开多个功能窗口，包括但不限于上述9个核心应用，以及计时器、错题本等辅助工具。
+ 窗口支持**拖拽、最小化、最大化、层叠**，状态自动保存。
+ 学生可保存多套工作区布局，例如"阅读模式"（讲义+笔记+导图）、"刷题模式"（习题+AI对话+计时器）——详见2.4.5工作区布局切换。

#### 2.4.3 上下文感知辅导

+ 在任意窗口内选中文本或代码，右键菜单或快捷键唤起 **AI辅导Agent**。
+ 辅导形式可包含：
    - 文字详细解释
    - 自动生成图解或流程图
    - 提供相似例题及分步解答
    - 生成简短的教学视频片段（加分项）
+ 解答内容将在新窗口或悬浮卡片中显示，不干扰当前学习流。

#### 2.4.4 关卡学习闭环与右键进入学习

**关卡学习闭环**

+ 每个关卡内部整合所有资源窗口，并提供明确的"完成关卡"按钮。
+ 点击完成后，系统自动收集本次学习行为数据（答题正确率、提问次数、阅读时长等），发送给画像Agent和评估模块。

**从藏宝图节点进入学习（零配置交互）**

+ 用户操作：在藏宝图中，右键单击任意一个关卡节点。
+ 系统响应：系统自动判断该关卡的知识类型（理论型 / 实践型含代码）：
  - **理论型关卡**：自动打开**讲义窗口 + 思维导图窗口 + 习题窗口**。三个窗口自动排列成预设的"学习布局"（例如讲义占左半，导图和习题叠在右半）。
  - **实践型关卡（含代码）**：自动打开**讲义窗口 + 代码实验室窗口 + 终端窗口**。代码实验室占主体，讲义和终端分居右侧和下侧。
+ 整个过程无需用户任何配置，右键后直接进入可学习状态。

**用户临时调整与智能记忆**

+ 如果系统自动打开的窗口多了：用户直接点击窗口右上角的"×"关闭即可，不影响学习。
+ 如果系统自动打开的窗口少了：用户从底部Dock栏点击对应图标，手动打开需要的窗口。
+ 系统会智能记忆用户偏好：例如用户连续三次在理论关卡中手动关闭了思维导图窗口，以后系统默认不再自动打开它。用户可在系统设置中一键"重置所有偏好"。

#### 2.4.5 工作区布局切换（一键切换 + 长按保存）

**预设布局图标**

在桌面顶部状态栏右侧放置三个小图标：

- 📖 阅读模式
- ✏️ 刷题模式
- 💻 代码实验模式

**切换操作（点击图标）**

+ 用户点击"📖 阅读模式"图标 → 当前所有已打开的窗口自动移动到预设位置（例如：讲义占左半边，笔记和导图叠在右半边，AI对话悬浮窗移到右下角）。
+ 如果某个预设布局需要的窗口当前没有打开（例如阅读模式需要笔记窗口，但笔记窗口没开），系统会自动打开该窗口。
+ 重要：窗口移动时，每个窗口里的内容完全不变（讲义还是刚才的讲义，习题还是刚才的习题，进度不丢失）。用户只是换了一个视角继续学习。

**自定义保存操作（长按图标）**

+ 用户手动调整好窗口位置、大小后（例如把讲义拉大、把AI对话移到左边），长按某个图标（如"📖 阅读模式"图标）2秒钟。
+ 系统弹出提示："是否将当前布局保存为阅读模式？"点击"是" → 当前布局覆盖该预设。
+ 下次点击该图标时，就会应用用户自定义的布局。

**交互示例**

1. 小明正在刷题，桌面窗口堆叠混乱。
2. 他点击顶部"✏️ 刷题模式"图标 → 习题窗口瞬间居中放大，AI对话浮窗移到右侧，计时器小窗出现在左上角。
3. 他觉得AI对话窗口应该在左边更顺手，于是手动把它拖到左边，然后长按"✏️ 刷题模式"图标2秒 → 系统问"保存为新的刷题模式？"点击确认。
4. 以后再点"✏️"图标，AI对话窗口就会自动出现在左边。

**一句话总结**：
- 进入学习：右键节点 → 自动开窗，不用配。
- 切换布局：点一下图标就换，长按图标就保存。

#### 2.4.6 我的资源库

**功能定位**
"我的资源库"是第8个核心应用窗口，允许用户上传自己的学习资料，并与系统内容生成流程深度整合，实现从"AI完全生成"到"人机共建"的升级。

**支持上传的文件类型**

+ PDF、Word、PPT、图片、Markdown、代码文件。

**上传后自动处理**

+ AI自动解析文件内容：对图片和扫描版PDF进行OCR识别，提取文字内容；对其他格式提取知识点摘要。
+ 解析结果以结构化形式存入资源库，支持搜索和预览。

**手动关联到关卡**

+ 用户可将资源库中的文件拖拽到左侧藏宝图的某个关卡节点上，完成绑定。
+ 绑定后，学习该关卡时，关联资料会出现在"拓展阅读"或"讲义附件"区域。

**AI生成优先引用**

+ 当用户学习已有关联资源的关卡时，内容生成Agent会优先引用用户上传的资料，生成个性化的讲义或习题，使学习内容更贴合用户已有的知识体系和语言风格。

**交互细节**

+ 支持批量上传、文件夹分类、文件标签（如"期末考试重点"、"课程论文"）。
+ 右键文件可选择"生成该文件的习题"或"总结该文件内容"，由AI即时处理。
+ 支持文件预览（PDF翻页、图片缩放、代码高亮），无需下载到本地即可查看。

---

### 2.5 学习效果评估与全局优化（加分项）
**功能描述**
基于多维度学习行为数据，对学生学习效果进行精准评估，并将评估结果反馈给画像和规划模块，实现学习方案的持续优化。

**评估维度**

+ **知识掌握度**：根据练习正确率、答题时间、是否反复查看答案等。
+ **能力进阶指数**：对比不同阶段关卡难度的通过情况，计算进步斜率。
+ **学习投入度**：有效学习时长、主动提问频率、资源浏览深度。
+ **弱点分布**：错误类型聚类（概念型、计算型、逻辑型）。

**评估反馈**

+ 评估报告以**仪表盘形式**呈现（独立窗口），包含趋势图、薄弱点雷达区等。
+ 仪表盘中增加**"画像演变"图表**：学生可选择时间范围（过去7天/30天/全部），查看每个画像维度数值的变化折线图。鼠标悬停在折线图的点上，显示当时的具体数值和触发事件（如"完成Transformer关卡后知识基础+0.1"）。
+ 评估结果驱动画像维度更新，并触发规划Agent调整后续关卡的内容、难度或顺序。
+ 可自动生成**阶段性学习总结报告**（PDF或Markdown），供学生下载或分享。

## **3. 非功能需求**
### 3.1 界面与交互规范
+ **风格**：简洁硬朗白色调，微小圆角，高信息密度，符合桌面应用专业感。
+ **响应式布局**：多窗口在PC上原生体验良好；适配平板或手机时，可切换为单页全屏模式。
+ **流式交互**：AI对话、资源生成均采用流式输出或进度条展示，避免长时间白屏。
+ **多模态内容呈现**：支持Markdown渲染、代码高亮、数学公式（LaTeX）、图片/视频卡片化展示。

### 3.2 性能与响应时间
+ 智能体核心对话响应首字时间 < 2秒。
+ 文档、习题、导图等纯文本资源生成总时长 < 10秒。
+ 视频/动画生成（加分项）可适当延长，但需提供任务队列与生成进度提示。
+ 系统支持至少50个并发学生使用而不明显降级。

### 3.3 内容安全与防幻觉
+ 必须内置**评审Agent**对生成内容进行事实核查与违规过滤。
+ 不允许出现违反学术伦理、敏感政治、侵权等不良信息。
+ 关键技术引用需标注来源（如使用开源模型、数据集）。

### 3.4 开放性与可扩展性
+ 遵循开源协议，使用开源项目时在文档显著位置声明。
+ 多智能体框架应支持方便地增加新Agent（如未来增加"口语陪练Agent"），通过注册中心即可接入。
+ 支持对接不同的LLM服务（讯飞星火、OpenAI等），通过统一接口适配。

## **4. 系统架构与多智能体设计**
### 4.1 整体架构
```plain
┌─────────────────────────────────┐
│          前端展示层              │
│  (React多窗口桌面 + 可视化库)   │
└──────────────┬──────────────────┘
               │ WebSocket / REST
┌──────────────┴──────────────────┐
│        API网关 & 消息队列       │
└──────────────┬──────────────────┘
               │
┌──────────────┴──────────────────┐
│       多智能体协同核心 (Python)  │
│                                 │
│  ┌────────┐ ┌────────┐ ┌──────┐│
│  │画像Agent│ │规划Agent│ │总监  ││
│  └────────┘ └────────┘ │Agent ││
│  ┌────────┐ ┌────────┐ └──┬───┘│
│  │文档子   │ │导图子   │   │    │
│  │Agent   │ │Agent   │←——┘    │
│  └────────┘ └────────┘        │
│  ┌────────┐ ┌────────┐        │
│  │习题子   │ │代码子   │        │
│  │Agent   │ │Agent   │        │
│  └────────┘ └────────┘        │
│  ┌────────┐                   │
│  │评审Agent│ (防幻觉/安全)     │
│  └────────┘                   │
└──────────────┬──────────────────┘
               │
┌──────────────┴──────────────────┐
│       数据层 & 知识库           │
│  画像数据库 / 关卡图谱 / 权威教材│
└─────────────────────────────────┘
```

### 4.2 Agent 详细职责
| Agent | 主要职责 | 输入 | 输出 |
| --- | --- | --- | --- |
| **画像Agent** | 对话引导、特征抽取、画像更新 | 学生对话历史、行为日志 | 更新后的画像JSON |
| **规划Agent** | 藏宝图生成、路径动态调整（含分支休眠/激活、模式切换） | 画像、课程目标、历史表现 | 关卡图谱数据 |
| **总监Agent** | 关卡资源清单制定、调度协调 | 关卡知识点、画像 | 资源生成任务列表 |
| **文档子Agent** | 讲义文档生成 | 知识点描述 | Markdown格式讲义 |
| **导图子Agent** | 思维导图生成 | 知识点关联列表 | 可交互导图数据 |
| **习题子Agent** | 分层练习题组生成 | 知识点、难度要求 | 题目列表+答案+解析 |
| **代码子Agent** | 实操案例/Notebook生成 | 代码场景描述 | 可执行代码 |
| **评审Agent** | 内容审查与防幻觉 | 各Agent输出原文 | 修改/驳回指令+安全日志 |

### 4.3 智能体协作流程（示例：关卡学习触发）
1. 学生右键点击藏宝图节点（或通过其他方式触发），前端发送 `start_level` 请求。
2. API网关将请求路由给**总监Agent**。
3. 总监Agent查询知识库获取该知识点元信息，结合画像确定资源参数（难度、学习模式、风格等），并创建任务：`generate_doc`, `generate_map`, `generate_exercise` 等。若该关卡绑定了用户上传的资源，总监Agent将资源引用一并传递给各子Agent。
4. 各子Agent并行生成，输出暂存至待审核区。
5. **评审Agent** 依次对每个输出进行评分和修正，通过则标记为 ready。
6. 前端通过 WebSocket 收到 `resource_ready` 事件，开始在对应窗口中流式渲染内容。
7. 若有审核未通过资源，总监Agent决定重试或降级处理。

## **5. 界面原型与交互设计**
### 5.1 主桌面概览
+ 顶部常驻菜单栏（可选，类macOS顶部栏），显示系统名称、时间，可呼叫全局搜索/AI对话框。
+ 顶部状态栏右侧放置：学习模式切换开关（🎯精通/🔭探索）和三个快捷布局图标（📖阅读 / ✏️刷题 / 💻代码实验）。
+ 桌面背景为纯色或极简纹理，摆放多个窗口。
+ 右下角固定一个**画像雷达图小窗口**（可隐藏），展示当前画像的6个维度，随学习过程实时刷新动画（闪光 + 数值增长，0.5秒内完成）。
+ 画像卡片上提供"编辑"按钮，学生可手动调整维度数值或标签。

### 5.2 关键窗口设计

**（1）藏宝图窗口**

+ 画布可缩放平移，节点用不同的圆形/矩形表示，连线表示依赖。
+ 工具栏：放大缩小、适应窗口、仅显示核心路径、导出图谱。
+ 右键节点可"标记重点""开始学习""查看详情"。
+ **分支休眠表现**：已休眠分支的连线和节点为浅灰色虚线，节点半透明并显示💤图标。悬停提示"此分支已休眠，可右键点击重新激活"。
+ **节点移动**：长按鼠标左键（>0.5秒）拖拽节点改变布局坐标，依赖关系不变；支持Ctrl多选批量拖拽。

**（2）讲义阅读窗口**

+ 类似PDF阅读器，内嵌Markdown渲染。
+ 可高亮文本并自动弹出"问AI"按钮。
+ 支持分页、夜间模式、字体调整。

**（3）习题练习窗口**

+ 题目卡片式展示，选择题、填空题、简答题均支持。
+ 提交后即时显示对错与解析，错题自动加入错题本。
+ 提供"再做一题类似题"按钮，由习题Agent动态生成。

**（4）AI对话悬浮窗**

+ 可在任何窗口顶层唤出，支持拖拽调整位置。
+ 对话记录关联当前学习上下文，具备记忆力。
+ 支持文字、代码块、图片、视频多种回复格式。

**（5）思维导图查看器**

+ 展示知识结构导图，支持节点展开/折叠。
+ 与讲义窗口联动：点击导图节点可跳转到讲义对应章节。
+ 支持导出为图片或 `.mm` 格式文件。

**（6）代码编辑器与运行终端**

+ 内置代码编辑器（支持语法高亮、自动补全），下方或右侧集成运行终端。
+ 支持Python、JavaScript等常见语言运行。
+ Jupyter Notebook模式可选，适配数据分析类关卡。

**（7）笔记本/批注工具**

+ 自由文本编辑区，支持Markdown格式。
+ 可在讲义内容上做高亮批注，批注自动归入笔记本。
+ 支持标签分类和全文搜索。

**（8）我的资源库**

+ 文件列表视图（列表/网格切换），显示文件名、类型、上传时间、关联关卡数。
+ 上传入口（拖拽上传或点击选择文件），支持批量上传和文件夹分类。
+ 文件预览面板：点击文件在右侧预览区域显示内容（PDF翻页、图片缩放、代码高亮）。
+ 右键菜单：生成习题、总结内容、关联到关卡、删除。
+ 搜索栏和标签过滤器。

**（9）学习仪表盘**

+ 数据概览卡片：总学习时长、完成关卡数、当前通过率。
+ 趋势图表：知识掌握度折线图、学习投入度柱状图。
+ **画像演变图表**：选择时间范围（7天/30天/全部），查看6个维度数值变化折线图。鼠标悬停显示具体数值和触发事件。
+ 薄弱点雷达图：聚类展示错误类型分布。

### 5.3 交互细节
+ 窗口最小化后缩为桌面图标，点击恢复。
+ 支持快捷键：`Ctrl+K` 唤起AI，`Ctrl+1/2/3` 切换工作区布局。
+ 右键藏宝图节点弹出上下文菜单，包含"开始学习"（自动开窗）、"标记重点"、"查看详情"、"放弃分支"（兴趣分支可用）、"重新激活"（休眠节点可用）。
+ 所有生成型操作均有loading动画，生成失败提供重试和降级文本。
+ 布局切换时窗口移动动画平滑（<0.3秒），内容不刷新不丢失。
+ 长按布局图标2秒触发保存确认弹窗。

## **6. 数据与知识库设计**
### 6.1 初始知识库构建
参赛团队需自建至少一门完整专业课程（如"大语言模型技术"）的初始知识库，格式可为结构化的JSON/Markdown文档集合，包含：

+ 知识点条目（名称、内容摘要、难度等级、前置知识点）
+ 教材引用文本、权威论文摘要
+ 典型例题与解答
+ 术语表

知识库作为生成时的参考源，保障内容权威性。

### 6.2 运行数据记录
+ 学生画像快照历史（含手动编辑记录）
+ 关卡学习记录（开始时间、结束时间、资源生成耗时、答题记录）
+ Agent协作日志（用于调试与优化提示词）
+ 评审Agent拦截记录
+ 用户资源库文件索引与关卡绑定关系
+ 用户偏好记录（自动开窗偏好、自定义布局配置）

## **7. 开发与部署规范**
1. **多智能体框架**：可使用 LangChain、AutoGen、CrewAI 等开源框架，或自研轻量Agent调度器。
2. **AI工具选型**：优先使用科大讯飞星火大模型API，其他辅助工具需符合赛题规定。
3. **前端技术**：建议 React + TypeScript，多窗口实现可用 `react-rnd` 或自研窗口管理器。
4. **部署**：提供 Docker 镜像及启动脚本，确保评委可在常规环境（如Ubuntu + NVIDIA GPU或CPU only）下运行。
5. **文档**：随系统提交完整的需求规格说明、系统设计、测试报告，详细阐述多智能体协作、防幻觉机制、前沿技术融合点。
