# 产品需求修订说明（v1）

## 1. 桌面布局：左侧地图 + 画像下置 + 右侧透明日历

### 1.1 原文引用

- 5.1 主桌面概览：右下角固定一个画像雷达图小窗口。
- 2.2：藏宝图在独立窗口中渲染，支持缩放、拖拽。

### 1.2 修订内容

主桌面采用全屏固定布局，分为左右两大区域：

- **左侧区域（占屏幕宽度的 1/3）**：进一步分为上下两部分。
  - 上部（占左侧区域高度的 2/3）：放置学习藏宝图。支持向上收起，收起后仅留左边缘的细把手；鼠标移到左边缘单击或向右拖拽即可下拉展开。
  - 下部（占左侧区域高度的 1/3）：放置用户画像卡片（雷达图 + 维度数值），常驻显示，不可收起。
- **右侧区域（占屏幕宽度的 2/3）**：放置半透明毛玻璃风格的日历组件。

### 1.3 解决拥挤的融合设计

- 左侧地图与画像卡片采用浅灰 / 白色底色，与桌面背景接近，卡片之间用极浅分割线区分。
- 右侧日历组件背景为毛玻璃模糊效果（`backdrop-filter: blur(20px)`），无硬边框，自然浮于桌面之上。
- 当地图收起时，右侧日历可向左扩展，同时画像卡片自动缩小为迷你雷达图图标（鼠标悬停展开完整卡片）。
- 用户可拖拽左侧上下区域的分割线，自定义地图与画像的高度比例。

### 1.4 右侧透明日历的交互

- 显示月历、每日任务列表、打卡状态、学习热力图（绿点深浅表示活跃度）。
- 支持点击日期查看当日详细任务，点击"今日打卡"按钮记录完成情况。
- 与左侧地图联动：
  - 右键单击地图节点可选择"添加到今日计划"；
  - 点击日历中的任务可自动展开地图并高亮对应节点。
- 支持拖拽日历任务到地图节点上，或拖拽地图节点到日历中生成任务。
- 日历可双击顶部边缘最小化为一个半透明小圆点（悬浮在右侧），鼠标悬停重新展开。

## 2. 藏宝图节点：被放弃分支 + 全局移动

### 2.1 原文引用

- 2.2：必须包含分支结构；兴趣分支（可选探索）。
- 未说明用户放弃分支后的处理方式，也没有节点位置移动的功能。

### 2.2 被放弃分支的视觉与交互

- 当用户长期未访问某兴趣分支，或主动选择"放弃此分支"时，该分支的连线和节点变为浅灰色虚线，节点半透明并显示小的"休眠"图标（💤）。
- 鼠标悬停时提示："此分支已休眠，可右键点击重新激活"。
- 右键单击休眠节点，弹出菜单"重新激活分支"，系统将其恢复为正常兴趣分支（银灰色实线 / 空心节点）。
- 分支被放弃后不会删除，仍保留在地图上，但默认不干扰主线视野。

### 2.3 节点全局移动

- 用户可长按鼠标左键（超过 0.5 秒）并拖动任意节点，改变其在画布上的视觉位置。
- 此操作仅改变节点的布局坐标，不改变知识图谱中的依赖关系（连线始终保持连接）。
- 支持配合 `Ctrl` 键多选节点后批量拖拽。
- 目的：允许用户自定义藏宝图的视觉布局，方便聚焦重点或整理个人学习路径。

## 3. 自适应难度控制：加入用户模式切换

### 3.1 原文引用

- 2.2.3.a：若学生快速通过某关卡，后续同类型关卡的难度自动提升。

### 3.2 修订内容

增加全局学习模式切换开关，位于桌面顶部状态栏右侧（或日历组件顶部），提供两个选项：

- 🎯 **精通模式**：系统按原文规则动态提升后续同类型关卡的难度。
- 🔭 **探索模式**：系统绝不提升难度，所有关卡按标准难度生成，适合快速了解、复习或初步学习。

### 3.3 交互与默认策略

- 默认首次使用为探索模式，用户可随时手动切换。
- 切换模式只影响未来新解锁的关卡，已生成的关卡内容及难度不变。
- 突击模式（原文已有）强制采用探索模式的难度策略，避免突击学习时难度突增。

## 4. 核心应用窗口清单与交互优化（含进入学习与布局切换）

### 4.1 原文引用

- 2.4.1 列出了：讲义阅读器、习题练习面板、AI 对话浮窗、代码编辑器与运行终端、笔记 / 批注工具。
- 2.3 提到了：知识结构导图、拓展阅读材料。
- 2.4.2：学生可保存多套工作区布局。

### 4.2 明确 9 个核心应用窗口

| # | 应用窗口 | 说明 |
| --- | --- | --- |
| 1 | 藏宝图 | 唯一，位于左侧侧栏 |
| 2 | AI 对话悬浮窗 | 可浮动或固定到某一侧 |
| 3 | 讲义阅读器 | 展示关卡内容 |
| 4 | 习题练习面板 | 练习与作答 |
| 5 | 代码编辑器 + 运行终端 | 代码实验与运行结果 |
| 6 | 笔记本 / 批注工具 | 个人笔记与标注 |
| 7 | 思维导图查看器 | 展示知识结构导图 |
| 8 | 我的资源库 | 用户上传的资料与 AI 解析摘要 |
| 9 | 学习仪表盘 | 学习数据统计与画像演变 |

### 4.3 从藏宝图节点进入学习（零配置交互）

**用户操作**：在左侧藏宝图中，右键单击任意一个关卡节点。

**系统响应**：系统自动判断该关卡的知识类型（理论型 / 实践型含代码）：

- **理论型关卡**：自动打开 **讲义窗口 + 思维导图窗口 + 习题窗口**。三个窗口自动排列成预设的"学习布局"（例如讲义占左半，导图和习题叠在右半）。
- **实践型关卡（含代码）**：自动打开 **讲义窗口 + 代码实验室窗口 + 终端窗口**。代码实验室占主体，讲义和终端分居右侧和下侧。

整个过程无需用户任何配置，右键后直接进入可学习状态。

**用户临时调整**：

- 如果系统自动打开的窗口多了：用户直接点击窗口右上角的"×"关闭即可，不影响学习。
- 如果系统自动打开的窗口少了：用户从底部 Dock 栏点击对应图标（如"笔记"），手动打开需要的窗口。
- 系统会智能记忆用户偏好：例如用户连续三次在理论关卡中手动关闭了思维导图窗口，以后系统默认不再自动打开它。用户可在系统设置中一键"重置所有偏好"。

### 4.4 工作区布局切换（一键切换 + 长按保存）

预设"快速布局"图标：在桌面顶部状态栏右侧放置三个小图标：

- 📖 阅读模式
- ✏️ 刷题模式
- 💻 代码实验模式

**切换操作（用户点击图标）**：

- 用户点击"📖 阅读模式"图标 → 当前所有已打开的窗口自动移动到预设位置（例如：讲义占左半边，笔记和导图叠在右半边，AI 对话悬浮窗移到右下角）。
- 如果某个预设布局需要的窗口当前没有打开（例如阅读模式需要笔记窗口，但笔记窗口没开），系统会自动打开该窗口。
- 重要：窗口移动时，每个窗口里的内容完全不变（讲义还是刚才的讲义，习题还是刚才的习题，进度不丢失）。用户只是换了一个视角继续学习。

**自定义保存操作（用户长按图标）**：

- 用户手动调整好窗口位置、大小后（例如把讲义拉大、把 AI 对话移到左边），长按某个图标（如"📖 阅读模式"图标）2 秒钟。
- 系统弹出提示："是否将当前布局保存为阅读模式？"点击"是" → 当前布局覆盖该预设。
- 下次点击该图标时，就会应用用户自定义的布局。

**交互示例**：

1. 小明正在刷题，桌面窗口堆叠混乱。
2. 他点击顶部"✏️ 刷题模式"图标 → 习题窗口瞬间居中放大，AI 对话浮窗移到右侧，计时器小窗出现在左上角。
3. 他觉得 AI 对话窗口应该在左边更顺手，于是手动把它拖到左边，然后长按"✏️ 刷题模式"图标 2 秒 → 系统问"保存为新的刷题模式？"点击确认。
4. 以后再点"✏️"图标，AI 对话窗口就会自动出现在左边。

**一句话总结**：

- 进入学习：右键节点 → 自动开窗，不用配。
- 切换布局：点一下图标就换，长按图标就保存。

## 5. 画像卡片显示与历史

### 5.1 原文引用

- 2.1：画像卡片窗口，雷达图展示，随学随新动画。
- 2.5：评估报告以仪表盘形式呈现，包含趋势图。

### 5.2 画像卡片显示

- 卡片位于左侧区域的下部（占左侧高度的 1/3），实时显示 6 个维度的雷达图及数值。
- 每次学习行为后（答对习题、完成关卡），对应维度有闪光 + 数值增长动画（0.5 秒内完成），用户可见变化。
- 卡片上提供"编辑"按钮，学生可手动调整某个维度的标签或数值（如将"注意力分散"改为"注意力集中"），系统更新画像。

### 5.3 画像历史查看

- 在【学习仪表盘】窗口中增加"画像演变"图表。
- 学生可选择时间范围（过去 7 天 / 30 天 / 全部），查看每个维度数值的变化折线图。
- 鼠标悬停在折线图的点上，显示当时的具体数值和触发事件（如"完成 Transformer 关卡后知识基础 +0.1"）。

## 6. 用户自传资料（我的资源库）

### 6.1 原文状态

原文未提及用户可上传自己的学习资料，本节为新增模块。

### 6.2 我的资源库窗口（第 8 个核心应用）

- 支持上传 PDF、Word、PPT、图片、Markdown、代码文件。
- 上传后 AI 自动解析文件内容（OCR 处理图片 / PDF），提取知识点摘要。

### 6.3 手动关联到关卡

- 用户可将资源库中的文件拖拽到左侧藏宝图的某个关卡节点上，完成绑定。
- 绑定后，学习该关卡时，关联资料会出现在"拓展阅读"或"讲义附件"区域。

### 6.4 AI 生成优先引用

- 当用户学习已有关联资源的关卡时，内容生成 Agent 会优先引用用户上传的资料，生成个性化的讲义或习题。

### 6.5 交互细节

- 支持批量上传、文件夹分类、文件标签（如"期末考试重点"）。
- 右键文件可选择"生成该文件的习题"或"总结该文件内容"，由 AI 即时处理。
